# Contributing

Agent OS Markdown is intentionally plain markdown. Contributions should make the operating model clearer, more testable, and more portable.

## Good Contributions

- sharpen a contract
- add a missing failure mode
- add an eval case
- clarify a permission boundary
- improve a template
- connect a doc to an implementation pattern

## Style

- Use direct language.
- Prefer checklists and contracts over slogans.
- Keep claims implementation-neutral.
- Do not hide uncertainty.
- Add examples when a rule could be misread.

## Change Checklist

- [ ] The change is useful outside one codebase.
- [ ] The change names the state it affects.
- [ ] The change has an observable verification path.
- [ ] The change does not encourage hidden side effects.
- [ ] The change keeps markdown readable in GitHub.
